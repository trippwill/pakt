PAKT

{{ site.Params.tagline }}

A typed data interchange format where every value carries its type.

app-name:str     = 'midwatch'
version:(int, int, int) = (2, 14, 0)
active:bool      = true
server:{host:str, port:int} = { 'localhost', 8080 }

# Streamed data — no delimiters needed
events:[{ts:datetime, level:str}] <<
    { 2026-06-01T14:30:00Z, 'info' }
    { 2026-06-01T14:31:00Z, 'warn' }
Read the Docs GitHub

Every value has a type

No inference, no ambiguity. Type annotations are producer assertions checked at parse time. Documents are self-validating.

Stream-parseable

Events emitted as data is read — no full-document buffering required. Process data as it arrives, not after it's all loaded.

Spec-projected

Consumer-side .spec.pakt files enable projections: materialize only the fields you need, skip everything else without allocation.

Performance is a feature

LL(0) streaming parser. Integer-typed events. No string dispatch. Zero lookahead for stream termination.

Why PAKT?

PAKT

server:{host:str, port:int} = {
    'localhost'
    8080
}

JSON

{
  "server": {
    "host": "localhost",
    "port": 8080
  }
}

Types are built in. Structure is explicit. Projections skip what you don't need.
